Why StoryAD-QA?

The StoryTeller paper studies long-form audio description for movies. A strong AD system should not only describe the current frame; it should preserve narrative context, resolve recurring characters, and explain why events matter as the story unfolds.

Standard captioning metrics are useful but limited: lexical overlap does not directly measure whether a generated AD contains the information needed to understand the story. StoryAD-QA was built to evaluate that missing dimension. It asks multiple-choice questions about actions, events, and narrative context, and scores whether a model's generated description supports the correct answer.

This release contains QA annotations, answer keys, prompts, and evaluation code. It does not include movie videos or other media assets.

Evaluation Code

The release includes a lightweight evaluator for multiple-choice accuracy. Predictions can be keyed by file identifier, row index, or both. 

pip install -r evaluation/requirements.txt
python evaluation/evaluate_storyad_qa.py \
  --gold data/StoryAD-QA/movie_qa_v5_30s_gemini-3-flash-preview.csv \
  --predictions predictions/my_model_v5_30s.csv

Prompts

We release the prompts used to create, regenerate, and verify StoryAD-QA questions so that the benchmark construction process is transparent and reproducible.

Generation and Regeneration

Templates for creating grounded five-choice QA items and rewriting failed questions.

Verification

A strict checklist for grounding, plausibility, correctness, and minor-fix decisions.

Relation To StoryTeller

StoryTeller is a training-free framework for long-form audio description with narrative memory. StoryAD-QA is released as the QA-based evaluation benchmark used to test whether AD outputs preserve story understanding. The benchmark complements lexical AD metrics by asking answerable questions about the generated narrative.
